🌉 What is the Königsberg Bridge Problem?

The Königsberg Bridge Problem is a famous historical problem in mathematics that laid the foundation for graph theory and topology.

The Problem: The city of Königsberg (now Kaliningrad, Russia) was situated on the Pregel River and included two large islands connected to each other and to the two mainland banks by seven bridges. The citizens wondered if it was possible to find a walk through the city that would cross every one of these seven bridges exactly once and return to the starting point.

What Is Web3

Jinghao Li 发布于 2025-11-28 收录于 好奇心  Web3

一、 互联网的架构演进：从“只读”到“所有权”

1. Web 1.0：只读网络 (Read-Only) —— “文件系统”

  • 本质： 互联网的早期阶段，本质上是一个全球共享的只读文件系统。
  • 技术逻辑： 用户只能被动地通过浏览器发送 GET 请求，服务器返回静态的 HTML 页面。内容由网站管理员单向发布，普通用户无法参与创作或修改，仅作为信息的消费者。

2. Web 2.0：读写网络 (Read + Write) —— “中心化数据库”

  • 本质： 平台经济时代，基于动态脚本（AJAX/JS）和中心化数据库（SQL）。
  • 技术逻辑： 用户不仅是消费者，也是创作者（UGC）。用户可以发帖、评论、交互，行为会实时通过后端 API 写入中央服务器的数据库中。
  • 痛点： 数据的所有权和管理权归属于平台（如腾讯、Google）。平台拥有最高权限（Root），可以随意删除用户数据或封禁账号，且用户创造的数据价值被平台垄断。

3. Web 3.0：价值网络 (Read + Write + Own) —— “分布式账本”

  • 本质： 基于密码学和共识机制的去中心化网络。
  • 技术逻辑： 核心数据（资产、合约逻辑）不再存储于单一的中心化服务器，而是通过区块链同步在全网节点上。
    • 去中心化： 没有单点故障，没有单一管理员。
    • 不可篡改： 依靠哈希链和共识算法，一旦数据上链，任何人都无法悄悄修改。
    • 所有权回归： 用户通过私钥（Private Key）直接控制自己的数字资产和身份，无需依赖第三方中介。
